### 任务背景

监管处罚数据是研究公司治理、信息披露质量和证券市场监管效果的重要数据来源。CSMAR RE 数据库收录了 1994 年以来沪深两市上市公司的违规及监管处罚全量信息，包含**违规信息总表**（STK_Violation_Main）和**违规信息明细表**（STK_Violation_Son）两张核心数据表。

然而，这份数据并不"干净"：违规类型和违规年度以特殊分隔符嵌套存储、日期字段用 `00` 表示缺失、主表与明细表存在一对多关系、部分字段含多重编码……这些特征使其成为练习数据清洗技能的绝佳素材。

本题的目标不是直接拿到干净数据进行分析，而是**亲手将一份结构复杂的原始数据转化为可分析的整洁面板**，在此过程中深刻理解真实数据的"脏"在哪里、为什么脏、该怎么处理。

#### 2.2 多值嵌套字段拆分（重点难点）

`ViolationTypeID` 和 `ViolationYear` 是结构最复杂的两个字段，两者**一一对应**，均以"年度间用分号分隔、同一年度内用顿号/逗号分隔"的方式存储。

以样本数据中的一条记录为例：

```
ViolationYear:     2019;2020;2021;2022;2023;2024;2025;N/A
ViolationTypeID:   P2504、P2505、P2599;P2599;P2599;P2503、P2599;...
```

**目标**：将上述一行记录拆分为多行，每行为"一个违规事件 × 一个违规年度 × 一种违规类型"的原子记录：

```
ViolationID  ViolationYear  ViolationTypeID  ViolationType
40216566     2019           P2504            推迟披露
40216566     2019           P2505            重大遗漏
40216566     2019           P2599            其他
40216566     2020           P2599            其他
40216566     2021           P2599            其他
...
```

实现步骤建议：

```python
def expand_violation(row):
    """
    输入：主表的一行（含 ViolationYear 和 ViolationTypeID）
    输出：拆分后的 DataFrame（多行）
    """
    vid = row["ViolationID"]
    years = str(row["ViolationYear"]).split(";")
    type_ids = str(row["ViolationTypeID"]).split(";")
```
