
请将我提供的 LaTeX / TeX 文档转换为 Quarto Markdown (.qmd) 格式。严格保留原文的实质内容、结构层级、数学公式、代码、图片、脚注、超链接、图表标题与交叉引用信息，不要擅自省略、改写、压缩、总结或润色正文内容。你的任务是“格式迁移 + 必要的轻度清理”，不是内容改写。

输出要求如下：

1. 输出结果必须适合直接保存为 `.qmd` 文件，并可用于 Quarto render 编译。
2. 不要添加 YAML 头。
3. 不要手工写章节编号，交给 Quarto 自动处理。
4. 删除同一段内部因排版造成的多余行尾回车，避免把一段误拆成多个段落。
5. 中英文混排时保留空格。
6. 数学公式保持 LaTeX 形式，不要改写为 Unicode 数学符号。
7. 若原文有 `\cite{...}`，先原样保留，不要擅自改成其它引文格式。

标题与标签转换规则：

1. `\chapter{...}` -> `# 标题`
2. `\section{...}` -> `## 标题`
3. `\subsection{...}` -> `### 标题`
4. `\subsubsection{...}` -> `#### 标题`
5. 若标题附近存在 `\label{sec:...}`，请将其并入标题行，写成：
   - `# 标题 {#sec-xxx}`
   - `## 标题 {#sec-xxx}`
   - `### 标题 {#sec-xxx}`
   - `#### 标题 {#sec-xxx}`

交叉引用转换规则：

1. 不要把 `\ref{...}` 替换为静态数字。
2. 请统一改为 Quarto 交叉引用语法：
   - `\ref{sec:xxx}` -> `@sec-xxx`
   - `\ref{eq:xxx}` -> `@eq-xxx`
   - `\ref{tab:xxx}` -> `@tbl-xxx`
   - `\ref{fig:xxx}` -> `@fig-xxx`
3. 若原标签不是标准前缀，但可明确判断其类型，请规范化：
   - 章节 -> `sec-...`
   - 公式 -> `eq-...`
   - 表格 -> `tbl-...`
   - 图片 -> `fig-...`
4. 若找不到对应标签，不要臆造编号，在正文中保留为 `[未解析引用: 原标签名]`，并在文末统一列出。

数学公式转换规则：

1. 行内公式保留为 `$...$`。
2. 独立公式统一转为 `$$ ... $$` 格式，公式块上下各空一行。
3. 对带 `\label{eq:...}` 的公式，不要手工写 `\tag{1}`，而应改为 Quarto 标签：
   ```qmd
   $$
   ...
   $$ {#eq-xxx}
````

4. 对 `equation*` 或无标签公式，保留为无标签 `$$ ... $$`。
5. 对 `aligned`、`split`、`gather`、`array` 等环境，尽量保留其数学结构并置于 `$$ ... $$` 内。
6. 对 `eqnarray*`，优先改写为 `aligned` 或等价结构。
7. 若原文显式使用 `\tag{A1}`、`\tag{*}` 等有实际含义的手工标签，则保留。

代码块转换规则：

1. `stverbatim` 或内部 `verbatim` 中的 Stata 代码，统一转为：

   ```stata
   ...
   ```
2. 其它 `verbatim` 环境转为 fenced code block；若语言无法判断，使用普通三反引号代码块。
3. 代码块上下各空一行，不要改写代码内容，不要把解释性文字混入代码块内部。
4. 若出现 `stlog` + `\input{...}`，且外部日志文件内容未提供，不要臆造，保留占位：

   ```text
   [外部日志文件未提供: 路径]
   ```

图片与表格转换规则：

1. 图片优先保留原文件名或相对路径，不要默认改成外部链接。
2. 图片改为 Quarto 图片语法，例如：

   ```qmd
   ![caption](figure.png){#fig-xxx width="80%"}
   ```
3. `\caption{...}` 转为图片标题，`\label{...}` 转为图片标签。
4. 图引用统一改为 `@fig-xxx`。
5. 简单表格可转为 Markdown pipe table，并添加 Quarto 表格标题与标签：

   ```qmd
   | A | B |
   |---|---|
   | ... | ... |

   : 表题 {#tbl-xxx}
   ```
6. 复杂表格可保留为 HTML / Pandoc 表格，不要为了纯 Markdown 化而破坏结构。

脚注与文本命令转换规则：

1. `\footnote{...}` 转为 Markdown / Pandoc 脚注：

   * 正文中使用 `[^1]`
   * 文末统一列出脚注内容
2. `\texttt{...}` -> `` `...` ``
3. `\textbf{...}` -> `**...**`
4. `\emph{...}` -> `*...*`
5. `\href{URL}{TEXT}` -> `[TEXT](URL)`
6. `\url{URL}` -> `<URL>`

清理规则：

1. 删除 `%` 开头的单行注释、行末注释，以及 `\iffalse ... \fi` 包围的整块注释内容。
2. 删除仅用于版面控制的 LaTeX 命令，例如：

   * `\setcounter{...}`
   * `\minitoc`
   * `\vspace{...}`
   * `\newpage`
   * `\nullskip`
3. 删除无意义的 `\begin{...}`、`\end{...}`、裸露 `\label{...}` 等 LaTeX 外壳。
4. 统一整理空行：

   * 标题前后保留合理空行
   * 公式块上下各空一行
   * 列表块上下各空一行
   * 代码块上下各空一行
   * callout 上下各空一行
   * 尽量避免连续多个空行

其它规则：

1. 若出现可明确解析的局部宏，如 `\def\FigName{...}`、`\def\Caption{...}`，请先展开再转换。
2. 若出现简单自定义数学宏，能可靠判断含义时可替换为标准 LaTeX 写法；不能可靠判断时，保留原状，并在文末统一列出。
3. 若发现重复 `\label{...}`，第一处保留原规范化标签，后续重复项重命名为 `-2`、`-3` 等后缀；若无法判断对应关系，则在正文中保留 `[未解析重复标签: ...]`，并在文末说明。
4. 对于“例子 / 注释 / 提示 / 警告”类自定义环境，优先转换为 Quarto callout。

最终输出要求：

1. 直接输出最终转换好的 `.qmd` 正文。
2. 不要解释处理过程。
3. 不要写“以下是转换结果”之类的说明。
4. 不要在正文中混入任何额外评注。
5. 若存在无法可靠解析之处，在全文末尾单独附一个 `## 未解析项目` 小节，统一列出：

   * 未识别宏
   * 缺失标签
   * 外部 `\input` 文件
   * 重复标签处理
   * 无法可靠转换的图表或环境
6. 除“未解析项目”外，不要添加任何说明性文字。
