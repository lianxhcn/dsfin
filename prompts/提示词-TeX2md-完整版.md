下面给你一份可直接复用的完整提示词。你以后把 `.tex` 文档和这段提示词一起发给另一个 AI，即可按同一套规则转换为 `.md` 文档。

# 可复用提示词

````text
请将我提供的 LaTeX / TeX 文档转换为 Markdown (.md) 格式。输出时严格遵循以下规则，不要擅自省略、改写或总结正文内容。你的任务是“格式转换 + 必要的轻度清理”，而不是改写内容。

## 1. 总体要求

1. 保留原文的全部实质内容、结构、层级、公式、代码、图片、脚注、超链接与提示框信息。
2. 不要压缩内容，不要概述，不要删减，不要重写学术表达。
3. 仅做以下几类处理：
   - LaTeX 结构转换为 Markdown 结构
   - 自定义环境与宏的替换
   - 公式编号与交叉引用的整理
   - 图片、脚注、超链接格式转换
   - 代码块语言标注与空行整理
   - 删除注释语句，即下面 '()' 中的内容：
     - `(^%.*)` 单行注释：删除
     - `{text} (%.*)` 行末注释
     - '\iffalse' 和 '\fi' 包围的蚊子块：删除
4. 若遇到以下情况，请先暂停并提问，不要擅自继续：
   - 无法识别的自定义宏
   - 无法判断含义的自定义环境
   - 表格结构过于复杂，可能在 Markdown 中失真
   - 图片文件名缺失或无法定位
   - 交叉引用缺少对应标签定义
5. 若大部分内容都可明确转换，则直接继续；仅将少数确实无法确定的地方集中列出。

## 2. 标题层级转换

将章节标题按如下规则转换，且标题不保留自动编号：

- `\chapter{...}` 转为一级标题：`# 标题`
- `\section{...}` 转为二级标题：`## 标题`
- `\subsection{...}` 转为三级标题：`### 标题`

若出现 `\subsubsection{...}`，默认转为四级标题：

- `#### 标题 {.unnumbered}`  ，即四级标题无需编号

除非我另行说明，否则不要保留 LaTeX 的章节编号。

## 3. 数学公式转换

### 3.1 行间公式

所有单行独立公式统一转为 `$$ ... $$` 格式，并满足以下要求：

1. 公式必须独占一行。
2. 公式上下各空一行。
3. 若原文是有编号公式，则在公式末尾加入 `\tag{#}`，其中 `#` 为该公式在全文中的顺序编号。

格式示例：

```md

$$
y = \alpha + \beta x + \varepsilon \tag{1}
$$

````

### 3.2 公式编号规则

1. 对原文中需要编号的单行公式，按在全文中首次出现的顺序依次编号：

   * `1`, `2`, `3`, ...
2. 若原文存在 LaTeX 标签，例如：

   * `\label{eq:xt-FWL-trend-01}`
     则你需要记录该标签与公式编号之间的映射关系。
3. 正文中凡是出现如下交叉引用：

   * `(\ref{eq:xt-FWL-trend-01})`
     应替换为：
   * `(1)`
4. 若出现 `式 \ref{...}`、`equation \ref{...}` 等类似写法，也替换成对应编号，例如：

   * `式 (1)`
   * `equation (1)`
5. 若某个 `\ref{...}` 找不到对应公式标签，请保留原标签名并显式标注：

   * `[未解析引用: eq:xxxx]`

### 3.3 数学宏替换

在数学公式中，执行以下自定义宏替换：

* `\x` 替换为 `\mathbf{x}`
* `\betaz` 替换为 `\boldsymbol{\beta}`

只替换这两个已明确指定的宏。若出现其它自定义数学宏，且无法确定含义，请先提问。

## 4. 列表格式要求

所有 list 格式块上下各空一行。包括：

* 无序列表
* 有序列表
* 嵌套列表

示例：

```md

- Item 1
- Item 2

```

不要让列表紧贴前后段落。

## 5. 代码块转换

### 5.1 `stverbatim` 环境

凡是 `stverbatim` 环境，一律转换为 Stata 代码块：

````md

```stata
... 原代码 ...
````

````

要求：

1. 代码块上下各空一行。
2. 不要改写代码内容。
3. 不要将代码解释混入代码块内部。

### 5.2 其它代码环境

若遇到明确可识别的代码环境，也尽量转为 fenced code block；如果语言无法判断，可使用普通三反引号代码块。但对于 `stverbatim`，必须使用 `stata` 语言标记。

## 6. 自定义提示框 `\boxtext` 的转换

将如下格式的自定义环境：

`\boxtext[...]{...}{...}{% ... }`

转换为 Quarto / Pandoc 风格的 callout 提示框，格式严格如下：

```md
::: {.callout-note}
### {title}
{text}
:::
````

处理规则：

1. `title` 使用 `\boxtext` 中表示标题的参数。
2. `text` 使用其正文内容。
3. 去掉 LaTeX 包装语法，仅保留标题和正文。
4. callout 前后各保留空行。

如果 `\boxtext` 的参数顺序在文中略有变化，请根据语义判断标题和正文。

## 7. 脚注转换

将 LaTeX 脚注：

* `\footnote{...}`

转换为 Markdown 脚注格式：

* 正文中插入 `[^1]`, `[^2]`, `[^3]`, ...
* 在全文末尾统一添加对应脚注内容

示例：

正文：

```md
这是一个结论。[^1]
```

文末：

```md
[^1]: 这里是脚注内容。
```

脚注编号按全文出现顺序依次编号。

## 8. 图片转换

将 LaTeX 图片转换为 Markdown 图片格式：

```md
![alt text](https://fig-lianxh.oss-cn-shenzhen.aliyuncs.com/{filename.png})
```

规则如下：

1. `{filename.png}` 必须使用原文中的完整图片文件名，包括后缀。
2. 图片链接统一写成：

   * `https://fig-lianxh.oss-cn-shenzhen.aliyuncs.com/{filename}`
3. `alt text` 若原文无明确说明，可使用简短描述或留空为：

   * `![image](...)`
4. 若原文图片带有 caption，可优先将 caption 用作 `alt text`。
5. 不要保留 LaTeX 的 `\includegraphics`、`figure` 等环境外壳。
6. 若存在 `\label{fig:...}`，可记录图标签；若正文有 `图 \ref{...}`，则尽量替换为自然表述，如：

   * `图 1`
     前提是图编号可以可靠识别。
7. 若图编号无法可靠恢复，则保留为普通图片，不强行编号。

## 9. 超链接与交叉引用转换

### 9.1 `\href`

将：

* `\href{URL}{Abowd et al. (1999)}`

转换为：

* `[Abowd et al. (1999)](URL)`

### 9.2 其它普通链接

若正文中存在普通 URL，也尽量转换为标准 Markdown 超链接，但不要改动链接文字含义。

### 9.3 `\ref` 与 `\label`

* 对公式引用：按“公式编号规则”处理
* 对章节、小节、图片引用：若能明确定位，则替换为自然文本，如“见下一节”“见图 2”“见第 3 节”
* 若无法可靠恢复，则保留标签名并显式标注：

  * `[未解析引用: sec:xxxx]`

## 10. 空行与版式规范

请统一整理 Markdown 版式，使其适合直接保存为 `.md` 文件：

1. 标题前后保留合理空行
2. 单行公式上下各空一行
3. 列表块上下各空一行
4. 代码块上下各空一行
5. callout 提示框上下各空一行
6. 脚注统一放在文末
7. 删除明显多余的 LaTeX 残留标记、空锚点、无意义注释
8. 不要保留多余的 LaTeX 命令外壳，如：

   * `\begin{...}`
   * `\end{...}`
   * `\label{...}`（除非用于引用映射）
   * 与排版无关的 LaTeX 注释
9. 尽量避免产生连续多个空行；一般保留一个空行即可

## 11. 输出要求

1. 直接输出最终转换好的 Markdown 正文。
2. 不要解释你的处理过程。
3. 不要写“以下是转换结果”之类的说明。
4. 不要在正文中混入额外评注。
5. 若存在少量无法确定之处，在全文末尾单独附一个简短的“未解析项目”小节，列出：

   * 未识别宏
   * 缺失标签
   * 无法确定的图或交叉引用
6. 除“未解析项目”外，不要添加其它说明性文字。

## 12. 执行顺序建议

请按以下顺序处理文档，以保证编号和引用一致：

1. 识别全文结构与标题层级
2. 提取并编号所有需要编号的公式，建立 `label -> 编号` 映射
3. 替换正文中的公式引用 `\ref{...}`
4. 转换数学公式为 `$$ ... \tag{#} $$`
5. 转换 `stverbatim` 为 ` ```stata `
6. 转换 `\boxtext` 为 callout
7. 转换脚注
8. 转换图片
9. 转换 `\href`
10. 清理空行、LaTeX 残留与版式

## 13. 特别提醒

1. 本任务是“忠实转换”，不是“内容改写”。
2. 对正文措辞、学术术语、公式符号、变量名、代码内容，尽量不作改动。
3. 除已明确指定外，不要擅自替换其它宏。
4. 若某些规则彼此冲突，请优先保证：

   * 内容完整
   * 公式编号正确
   * Markdown 可直接使用


注意：在 .tex 文档中是使用空行来分段的，你转换成 Markdown 时要把 .tex 中同一段内多个句子的行尾回车符删掉，否则转后的 .md 文档会有很多个段落。

```

你以后可以这样配套使用：

- 先贴上面这段提示词
- 再附一句：`下面是待转换的 .tex 文档内容，请开始转换。`
- 再粘贴 `.tex` 原文

```
