
请将我提供的 LaTeX / TeX 文档转换为 Quarto Markdown (.qmd) 格式。输出时严格遵循以下规则，不要擅自省略、改写、压缩、总结或润色正文内容。你的任务是“格式迁移 + 必要的轻度清理”，而不是内容改写。

## 1. 总体目标

1. 保留原文的全部实质内容、结构层级、数学公式、代码、图片、脚注、超链接、图表标题、交叉引用与提示框信息。
2. 输出结果必须适合直接保存为 `.qmd` 文件，并可用于 Quarto render 编译。
3. 优先保留 Quarto 的自动编号与交叉引用体系，不要将 LaTeX 的 `\ref` 机械替换为静态数字，除非我明确要求。
4. 不要概述，不要删减，不要重写学术表达，不要修改变量符号、术语含义和代码逻辑。
5. 若大部分内容可以明确转换，则直接完成转换；只有少量无法确定之处，统一放在文末“未解析项目”小节中列出，不要频繁中断。
6. 遇到外部文件未提供、局部宏无法解析、标签重复、复杂表格无法可靠迁移等情况时，不要臆造内容，要保留占位并在文末说明。
7. 注意：`.tex` 文档中同一段内部常因排版而被人工换行。转换为 `.qmd` 时，请删除同一段内部多余的行尾回车，避免把一段误拆成多个段落。
8. 不要添加 YAML 头。
9. 不要手工写章节编号，交给 Quarto 自动处理。
10. 中英文混排时保留空格。
11. 数学公式保持 LaTeX 形式，不要改写为 Unicode 数学符号。
12. 若原文中有 `\cite{...}`、`\citet{...}`、`\citep{...}` 等引文命令，默认先原样保留，不要擅自改成其它引文格式，除非我另行要求。

## 2. 输出格式与引用体系

本次目标格式是 `.qmd`，请遵循 Quarto / Pandoc 的标签与交叉引用语法，而不是普通 Markdown 的静态写法。

统一使用如下标签前缀：

- 章节标签：`{#sec-xxx}`
- 公式标签：`{#eq-xxx}`
  - 公式标签要紧接在公式块末尾，格式为 `$$ ... $$ {#eq-xxx}`，不要放在公式的下一行。
- 图片标签：`{#fig-xxx}`
- 表格标签：`{#tbl-xxx}`

正文中的交叉引用统一使用：

- `@sec-xxx`
- `@eq-xxx`
- `@fig-xxx`
- `@tbl-xxx`

不要把这些引用提前替换成固定数字编号。

## 3. 标题层级转换

将标题转换为 Quarto 标题，默认规则如下：

- `\part{...}` -> `# 标题`
- `\chapter{...}` -> `# 标题`
- `\section{...}` -> `## 标题`
- `\subsection{...}` -> `### 标题`
- `\subsubsection{...}` -> `#### 标题 {.unnumbered}`

若标题附近存在 `\label{sec:...}`，请将标签并入标题行，格式如下：

- `# 标题 {#sec-xxx}`
- `## 标题 {#sec-xxx}`
- `### 标题 {#sec-xxx}`
- `#### 标题 {#sec-xxx}`

标签规范化规则：

- `sec:GLM-intro` -> `sec-GLM-intro`
- `sec:prob-basic-01` -> `sec-prob-basic-01`
- `eq:prob-basic-01` -> `eq-prob-basic-01`
- `tab:prob-basic-01` -> `tbl-prob-basic-01`
- `fig:example-01` -> `fig-example-01`

若原图标签不是 `fig:` 前缀，但可明确判断其是图片标签，也请规范化为 `fig-...`，例如：

- `Logit-Link-function-01` -> `fig-Logit-Link-function-01`

不要手工写章节编号，如“## 1.2 ...”；交给 Quarto 自动处理。

## 4. 交叉引用转换

### 4.1 基本规则

将 LaTeX 引用转换为 Quarto 引用：

- `\ref{sec:GLM-intro}` -> `@sec-GLM-intro`
- `\ref{eq:prob-basic-01}` -> `@eq-prob-basic-01`
- `\ref{tab:prob-basic-01}` -> `@tbl-prob-basic-01`
- `\ref{fig:example-01}` -> `@fig-example-01`

若原标签不是标准前缀，但上下文可明确判断其类型，请转换为相应的 Quarto 引用：

- 章节 -> `@sec-...`
- 公式 -> `@eq-...`
- 表格 -> `@tbl-...`
- 图片 -> `@fig-...`

### 4.2 行文中的自然替换

保留自然语言，但将引用本身改成 Quarto 语法。例如：

- `见式 \ref{eq:prob-basic-01}` -> `见 @eq-prob-basic-01`
- `图 \ref{fig:example-01}` -> `@fig-example-01`
- `第 \ref{sec:GLM-intro} 节` -> `@sec-GLM-intro`

不要替换成“见下一节”“见上图”“见图 2”这类静态表述。

### 4.3 `\eqref` 的处理

将 `\eqref{eq:xxx}` 也改为 `@eq-xxx`。不要保留 LaTeX 风格的括号编号格式，交给 Quarto 自动渲染。

### 4.4 找不到标签时的处理

若某个 `\ref{...}`、`\eqref{...}` 找不到对应标签：

- 不要臆造编号；
- 在正文中保留为 `[未解析引用: 原标签名]`；
- 并在文末“未解析项目”中列出。

## 5. 数学公式转换

### 5.1 行内公式

行内公式保留为 `$...$` 格式，不要改写数学含义。

### 5.2 行间公式

独立公式统一转换为 `$$ ... $$` 格式。公式块上下各空一行，公式必须独占一行。

### 5.3 有编号公式

对于原文中带 `\label{eq:...}` 的公式，不要手工写 `\tag{1}`，而应使用 Quarto 标签。例如：

```qmd
$$
\mathrm{E}\left(y_i \mid \mathbf{x}_i\right)=G\left(\mathbf{x}_i;\beta\right)
$$ {#eq-prob-basic-01}
````

### 5.4 无编号公式

对于 `equation*`、无标签公式、纯展示性公式，保留为无标签 `$$ ... $$`。

### 5.5 多行公式环境

请按以下规则处理：

* `equation + aligned`：保留为一个 `$$ ... $$ {#eq-xxx}`，内部保留 `\begin{aligned}...\end{aligned}`
* `split`：保留在 `$$ ... $$` 内部
* `aligned`：保留在 `$$ ... $$` 内部
* `gather` / `gather*`：转换为单个 `$$ ... $$` 块，内部尽量保留多行结构
* `multline` / `multline*`：转换为单个 `$$ ... $$` 块，尽量保留换行结构
* `array`：若用于分段函数、矩阵、分情况表达，可保留在公式内部
* `eqnarray` / `eqnarray*`：优先改写为 `aligned` 或等价结构，不要继续保留 `eqnarray`
* `cases`：保留在公式内部
* `matrix`、`pmatrix`、`bmatrix` 等矩阵环境：保留在公式内部

### 5.6 手工 `\tag`

若原文显式使用 `\tag{A1}`、`\tag{*}`、`\tag{2.3}` 等有实际含义的手工标签，则保留原 `\tag{...}`。但若只是普通自动编号公式，不要人为补写 `\tag{1}`、`\tag{2}`。

### 5.7 公式标签规范化

将原 `\label{eq:xxx}` 转为公式块末尾的 `{#eq-xxx}`。不要在正文中残留裸露的 `\label{...}`。

## 6. 数学宏与自定义命令

### 6.1 已知宏

若文中存在已知自定义数学宏，请按语义替换。例如：

* `\x` -> `\mathbf{x}`
* `\betaz` -> `\boldsymbol{\beta}`

### 6.2 其它简单数学宏

若还出现其它可可靠推断含义的简单宏，例如：

* `\X`
* `\y`
* `\thetaz`
* `\eps`
* `\Var`
* `\Cov`

请在不改变原意的前提下替换为标准 LaTeX 写法。若无法可靠判断，则保留原状，并在“未解析项目”中列出。

### 6.3 局部文本宏

若在局部代码块、图表环境附近出现如下定义：

* `\def\FigName{...}`
* `\def\Caption{...}`
* `\def\TabName{...}`

请先解析这些局部宏，再完成图表转换。最终输出中不要残留未展开的 `\FigName`、`\Caption`、`\TabName` 之类占位宏。

### 6.4 无法识别的复杂宏

若遇到复杂自定义命令、依赖导言区宏包实现的特殊语法，且无法可靠推断含义：

* 不要擅自改写；
* 尽量保留原文片段；
* 在文末“未解析项目”中列出宏名和上下文。

## 7. 列表格式

所有列表块上下各空一行，包括：

* 无序列表
* 有序列表

## 8. 代码块转换

### 8.1 `stverbatim`

若出现：

* `\begin{stverbatim} ... \begin{verbatim} ... \end{verbatim} ... \end{stverbatim}`

请提取最内层代码内容，转换为：

```stata
... 原代码 ...
```

代码块上下各空一行，不要改写代码内容，不要把解释性文字混入代码块内部。

### 8.2 `verbatim`

普通 `verbatim` 环境也转为 fenced code block。若语言无法判断，使用普通三反引号代码块；若明显是 Stata 代码，则使用 `stata`。

### 8.3 其它代码环境

若遇到明确可识别的代码环境，也尽量转为 fenced code block：

* Stata -> `stata`
* Python -> `python`
* R -> `r`
* Bash / Shell -> `bash`

若无法可靠识别语言，则使用普通三反引号代码块。

### 8.4 `stlog`

若出现：

* `\begin{stlog} \input{sjlog/xxx.log.tex}\nullskip \end{stlog}`

处理规则如下：

1. 若外部日志内容已在我提供的材料中给出，则内联展开并转为合适的代码块或普通文本块。
2. 若外部日志内容未提供，则不要臆造输出，改为占位块，例如：

```text
[外部日志文件未提供: sjlog/xxx.log.tex]
```

3. 同时在文末“未解析项目”中记录该外部文件路径。

### 8.5 `\input{...}` 和 `\include{...}`

若正文中出现外部输入文件：

* 若其内容已提供，可内联展开；
* 若未提供，不要臆造，保留占位并在文末说明。

### 9.2 `\boxtext`

### 9.3 其它自定义环境

若能明确判断为“例子 / 注释 / 提示 / 警告 / 说明 / 定义 / 结论”类块级内容，优先转换为 Quarto callout。若无法判断，则保留正文内容，并在文末说明环境名。

## 10. 图片转换

### 10.1 基本原则

优先保留原图片文件名或相对路径，不要默认改成外部链接，除非我明确要求。

### 10.2 Quarto 图片语法

将 LaTeX 图片转换为 Quarto 图片语法。例如：

```qmd
![LPM，Probit 和 Logit 模型的链接函数](Logit-Link-function-01.png){#fig-Logit-Link-function-01 width="100%"}
```

### 10.3 宽度转换

将 LaTeX 中常见的图片宽度转换为 Quarto 百分比：

* `width=.8\textwidth` -> `width="80%"`
* `width=0.8\textwidth` -> `width="80%"`
* `width=1.0\textwidth` -> `width="100%"`
* `width=\textwidth` -> `width="100%"`

若宽度无法可靠判断，可省略宽度属性。

### 10.4 `figure` 环境

若图片原本位于 `figure` 或相关环境中，请提取：

* 图片文件名
* caption
* label
* width 等属性

不要保留 `figure` 环境外壳。

### 10.5 图片标签与标题

* `\caption{...}` -> 图片标题
* `\label{...}` -> 图标签
* 局部 `\Caption` / `\FigName` 先展开再转换

### 10.6 图引用

正文中的图引用统一改为 `@fig-xxx`。

### 10.7 多图结构

若原文是多图并排、subfigure、subcaption 结构：

* 尽量用 Quarto / Markdown 可表达的方式保留结构；
* 若无法完整复现，也不要随意打散其语义关系；
* 可保留为 HTML 结构或简单顺序图块；
* 文末注明“子图结构未完全保留”。

## 11. 表格转换

### 11.1 简单表格

若 LaTeX 表格结构简单，可转为 Markdown pipe table，并添加 Quarto 表格标题与标签。例如：

```qmd

| ... | ... | ... |

: 条件相对频率分布 {#tbl-prob-basic-01}
```

### 11.2 复杂表格

若表格过于复杂，不适合 pipe table，请优先：

1. 保留为可读性更高的 Pandoc / HTML 表格；
2. 不要为了“纯 Markdown 化”而破坏结构；
3. 在表格标题中保留 `tbl-` 标签；
4. 文中引用改为 `@tbl-xxx`。

### 11.3 表格标题与标签

* `\caption{...}` -> 表题
* `\label{tab:...}` -> `tbl-...`

### 11.4 表格引用

正文中的表格引用统一改为 `@tbl-xxx`。

## 12. 脚注转换

将 `\footnote{...}` 转为 Pandoc / Markdown 脚注格式：

* 正文中使用 `[^1]`、`[^2]`、`[^3]`
* 文末统一列出脚注内容

脚注编号按全文首次出现顺序依次排列。

若存在脚注中的公式、链接、强调命令，请尽量保留其原始语义。

## 13. 常见文本命令转换

尽量做如下轻度转换：

* `\texttt{...}` -> `` `...` ``
* `\textbf{...}` -> `**...**`
* `\emph{...}` -> `*...*`
* `\href{URL}{TEXT}` -> `[TEXT](URL)`
* `\url{URL}` -> `<URL>`
* `\underline{...}` -> 视语义决定是否保留原文或转换为普通文本
* `\item` -> 转为列表项
* `\\` 若用于普通段落换行，尽量改为自然段落或合适的 Markdown 换行；若位于数学环境内则保留

不要因转换这些命令而改变语义。

## 14. 引文与参考文献

1. 对 `\cite{...}`、`\citet{...}`、`\citep{...}`、`\citealp{...}` 等引文命令，默认先原样保留，不要擅自改写为 Pandoc 引文格式。
2. 若正文末尾有 `thebibliography` 环境，可先原样保留为纯文本参考文献块，或按 Markdown 列表输出，但不要删减条目。
3. 若参考文献依赖 `.bib` 文件而未提供，不要臆造具体文献内容。

## 15. 注释与无效块处理

删除以下内容：

* `%` 开头的单行注释
* 行末注释
* `\iffalse ... \fi` 包围的整块注释内容

注意：删除注释时，不要误删正文中的 `%` 字符本义。

## 16. 仅用于排版的 LaTeX 命令

以下命令通常应删除，不必保留到 `.qmd`：

* `\setcounter{...}`
* `\minitoc`
* `\vspace{...}`
* `\hspace{...}`（若仅用于版面微调）

但若删除会影响语义或结构识别，则谨慎保留。

## 17. 重复标签处理

若发现重复 `\label{...}`：

1. 第一处保留原规范化标签；
2. 后续重复项重命名为：

   * `eq-xxx-2`
   * `eq-xxx-3`
   * `fig-xxx-2`
   * `tbl-xxx-2`
   * `sec-xxx-2`
3. 同步更新其就近引用；
4. 若无法判断某处引用应指向哪一个，则在正文中保留 `[未解析重复标签: 原标签名]`，并在文末说明。

## 18. 空行与版式规范

请统一整理 `.qmd` 版式，使其适合直接保存与编译：

1. 标题前后保留合理空行。
2. 行间公式上下各空一行。
3. 列表块上下各空一行。
4. 代码块上下各空一行。
5. callout 上下各空一行。
6. 脚注统一放在文末。
7. 删除明显多余的 LaTeX 残留标记、空锚点和无意义注释。
8. 不要保留多余的 `\begin{...}`、`\end{...}`、裸露 `\label{...}` 等 LaTeX 外壳。
9. 尽量避免连续多个空行，一般保留一个空行即可。

## 19. 未解析项目

若存在无法可靠解析之处，请在全文末尾单独附一个：

```markdown
## 未解析项目
```

* 未识别宏
* 缺失标签
* 外部 `\input` / `\include` 文件

## 20. 输出要求

## 21. 执行顺序

请严格按以下顺序处理：

1. 识别标题层级与 section 标签。
2. 提取全部 `\label`、`\ref`、`\eqref`，建立映射。
3. 检测重复标签。
4. 解析局部宏，例如 `\FigName`、`\Caption`、`\TabName`。
5. 处理公式环境，包括 `equation`、`aligned`、`split`、`gather`、`multline`、`array`、`eqnarray*`。
6. 将引用改为 Quarto 语法 `@...`。
7. 处理图片与表格。
8. 处理代码块，尤其是 `stverbatim`、`verbatim`、`stlog`、`\input`。
9. 处理脚注、超链接、文本格式命令。
10. 清理注释、排版命令与多余空行。
11. 在末尾附“未解析项目”。
