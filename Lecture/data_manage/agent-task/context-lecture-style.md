# 讲义写作风格参考

## 核心风格原则

本课程讲义（第一章「金融数据获取」、第三章「数据清洗」）有一套独特的写作风格。
第二章必须与之保持一致。以下是关键特征：

---

## 1. 语气与定位

**对象：** 金融硕士研究生，有一定 Python 基础，目标是进入金融机构从事数据分析。

**语气：** 直接、务实、不绕弯子。不追求学术严谨，追求"讲清楚为什么要这么做"。

**典型句式风格（来自第一章）：**
- "听起来像废话，但很多人在这一步就卡住了"
- "这种做法在项目初期似乎没有问题，但随着项目持续推进，问题会迅速暴露出来"
- "你会发现，不管是...还是...，代码结构都是这个样子"
- "不需要死记，用到哪类数据时查一下官方文档或让 AI 帮你找就行"

**禁忌：**
- 不说"首先...其次...最后..."这种套话
- 不用"值得注意的是"、"不难发现"这类学术腔
- 不追求面面俱全，宁缺毋滥

---

## 2. 代码注释风格

全部中文注释，风格参考第一章：

```python
# 获取全部 A 股实时行情（约 5000 只股票）
df_spot = ak.stock_zh_a_spot_em()

# 设置 User-Agent，模拟浏览器访问，大多数服务器不会拒绝这种请求
headers = {'User-Agent': 'Mozilla/5.0'}

# 如果服务器返回错误（如 404 Not Found、403 Forbidden），
# raise_for_status() 会直接抛出异常，便于快速定位问题
resp.raise_for_status()
```

特点：
- 注释说"为什么"，不只说"是什么"
- 对容易出错的地方加警示注释
- 括号里补充背景信息（约XXX条、格式YYYYMMDD等）

---

## 3. 章节结构

每节（Section）的标准结构：

```
## Sec X  节标题

[1-2段引导文字：为什么要讲这个，解决什么问题]

[概念解释或对比表格]

[代码演示，配注释]

[输出结果说明]

（可选）[进阶提示或常见错误]

::: {.callout-tip}
### AI 提示词：[具体场景]
[提示词模板]
:::
```

---

## 4. Callout 格式（Quarto 语法）

```markdown
::: {.callout-tip}
### AI 提示词：XXX
提示词内容
:::

::: {.callout-note}
### 注意
内容
:::

::: {.callout-warning}
### 常见错误
内容
:::
```

第一章和第三章都用了这套格式，第二章保持一致。

---

## 5. 数学公式

用 LaTeX，行内用 `$...$`，行间用 `$$...$$`。
第三章开头用了：$$\text{analysis quality} = f(\text{data quality}, \text{data organization})$$
这类公式不要过度使用，只在真正有助于表达的地方用。

---

## 6. 表格

用 Markdown 表格，对齐整洁。对比类内容（格式对比、pandas vs SQL）优先用表格而非列表。

---

## 7. 真实数据优先

第一章的所有示例都用真实数据（akshare 拉取的真实股票、真实宏观数据），
第三章用真实研究场景（上市公司借款利率）。第二章同样要以真实数据为主，
mini-dataset 仅用于概念演示，篇幅控制在全章的 1/3 以内。

---

## 8. 一个可以直接参考的段落（第一章原文）

> 这四步的顺序有其内在逻辑：先知道「有多少」，再看「长什么样」，接着确认「类型和完整性」，最后检查「数值是否合理」。每一步都有可能暴露出问题。

这段话的特点：
- 说清楚了"为什么是这个顺序"
- 用了具体动词（知道、看、确认、检查）
- 不废话，但也不干巴巴
- 最后一句有画龙点睛的效果

写第二章时，类似的说理段落要保持这个密度和风格。
