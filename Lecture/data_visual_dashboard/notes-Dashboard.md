## 讲义说明

### 1. 如何理解 Dashboard？

Dashboard 可译为“数据仪表板”或“可视化看板”，其核心目标不是展示尽可能多的图表，而是围绕一个明确问题，将关键指标、趋势、结构和异常集中展示，帮助使用者快速监测、比较、判断和决策。

对于金融场景而言，Dashboard 往往服务于以下任务：

- 市场监测：如股票指数、行业轮动、成交额、波动率变化
- 组合跟踪：如收益率、回撤、夏普比率、资产配置权重
- 风险管理：如 VaR、最大回撤、敞口、久期、信用分层
- 业务分析：如银行客户结构、券商业务条线、地区融资分布
- 政策与宏观跟踪：如利率、汇率、通胀、社融、地产与能源数据

因此，Dashboard 本质上是“面向决策的问题组织方式”，而不只是“若干张图的拼接”。

### 2. 原始工具包名单的筛选结论

你给出的工具包名单里，确实有一部分是金融领域常用工具，但它们的用途并不相同，也并非都适合作为这门课中“Dashboard”主题的核心内容。更准确地说，可以分成以下几类。

#### （1）适合保留：与金融分析、绩效评价、回测直接相关

- **QuantStats**  
  适合做策略绩效分析、收益与风险评估，能够快速输出较完整的分析报表。对于教学非常友好，适合作为“金融分析结果可视化”的示例工具。

- **Pyfolio**  
  经典的投资组合绩效分析工具，适合讲解收益、波动、回撤、滚动指标等内容。虽然近年维护活跃度不如以前，但教学参考价值仍然较高。

- **Empyrical**  
  更偏底层指标计算库，常与 Pyfolio 配合使用。适合介绍金融绩效指标的计算逻辑，如累计收益、年化收益、夏普比率、最大回撤等。

- **Alphalens**  
  适合做因子分析、分组收益、IC 分析、换手率分析等。若课程涉及因子投资、量化选股或截面预测，这个工具非常值得保留。

- **Backtrader**  
  Python 中较经典、易上手的回测框架，文档多、案例多，适合作为教学演示平台。

- **bt**  
  语法相对简洁，适合做资产配置和规则型策略回测。对初学者比较友好。

#### （2）建议弱化：可以提及，但不作为主讲重点

- **Zipline**  
  历史上很有代表性的回测框架，但对环境和数据适配要求较高，对中国市场的直接支持不够友好。可以作为“了解过的经典框架”提一下，但不建议作为课程主工具。

- **QuantConnect**  
  它本质上更像一个量化研究与交易平台，而不是单纯的 Python 包。优点是平台化、多市场、多策略、云端运行；缺点是离中国本土数据生态较远。适合作为“综合平台介绍”补充，不建议作为本课程主线。

- **QuantLib**  
  这是金融工程和衍生品定价领域非常重要的库，但它更偏利率、债券、期权、曲线构建和定价模型，不属于 Dashboard 课程的核心工具。可以在讲到固定收益、衍生品或金融工程时再重点展开。

#### （3）不建议纳入本讲 Dashboard 重点

- **FinRL**  
  它主要面向强化学习驱动的量化交易研究，门槛较高，依赖较复杂，更适合机器学习或智能投顾专题，不适合作为 Dashboard 入门或常规金融可视化课程的核心工具。

### 3. 更适合本课程的工具包组织方式

考虑到你的授课对象是金融专业和数字经济专业硕士研究生，其中不少同学来自券商、银行、政府、南方电网等单位，因此课程工具应同时满足：

- 尽量容易安装和上手
- 与中国市场数据衔接顺畅
- 能支撑金融分析和业务展示
- 便于后续做课程作业和案例复现

基于这个目标，更推荐按下面 5 个层次组织工具。

#### 第一层：数据获取工具（中国市场优先）

- **Tushare**  
  A 股、基金、宏观、财务、行业等数据接口丰富，教学中使用非常广泛。  
  适合：股票、基金、财务指标、宏观金融数据获取。

- **AkShare**  
  覆盖股票、期货、债券、宏观、新闻等多类数据源，接口风格统一，社区使用广泛。  
  适合：快速抓取中国市场与宏观数据，做课堂演示非常方便。

- **WindPy**（如机构或学校有授权）  
  若学生来自券商、银行或高校实验室，Wind 数据往往更贴近真实业务场景。  
  适合：高质量金融终端数据接入与机构场景演示。

- **iFinDPy / 同花顺 API**（如机构可用）  
  对部分机构用户也很实用，但教学普适性不如 Tushare 和 AkShare。

> 结论：若面向普遍教学，优先推荐 **Tushare + AkShare**。

#### 第二层：通用数据分析与可视化工具

- **pandas**：金融时间序列、面板数据、分组统计的核心工具  
- **numpy**：数值计算基础  
- **matplotlib**：最基础、最稳定的绘图库  
- **seaborn**：更适合统计图、分布图、热力图  
- **plotly**：适合交互式图表  
- **streamlit / dash**：适合快速搭建交互式 Dashboard 原型

这部分工具不专属于金融，但它们才是实际做 Dashboard 时最常用、最稳定的底层能力。

> 结论：若讲“Dashboard 实作”，这一层反而是主干。

#### 第三层：金融绩效分析与因子研究工具

- **QuantStats**：快速生成策略绩效分析页面  
- **Pyfolio**：投资组合绩效分析经典工具  
- **Empyrical**：绩效指标计算底层库  
- **Alphalens**：因子检验与分层分析工具

这类工具非常适合讲“金融专业特色”，因为它们直接对应收益、风险、绩效评价与因子研究。

#### 第四层：回测框架

- **Backtrader**：最推荐，适合教学与案例演示  
- **bt**：轻量、简洁，适合资产配置类策略  
- **Zipline**：了解即可，不建议作为主力框架

#### 第五层：综合平台与进阶工具

- **QuantConnect**：适合介绍云端量化平台的概念和多市场研究框架  
- **QuantLib**：适合在固定收益、衍生品定价课程中扩展介绍  
- **FinRL**：适合机器学习/强化学习专题，不建议在本讲作为重点

### 5. 可直接用于课堂的讲法建议

若你的目标是“让硕士生在一讲之内形成可操作的 Dashboard 框架”，可以按下面的逻辑来讲：

1. **先讲 Dashboard 是什么、解决什么问题**
2. **再讲中国市场数据从哪里来**
   - Tushare
   - AkShare
3. **再讲怎样用 pandas + matplotlib/seaborn/plotly 做核心图表**
4. **再讲金融场景下的专用分析包**
   - QuantStats
   - Pyfolio
   - Empyrical
   - Alphalens
5. **最后补充回测框架和综合平台**
   - Backtrader、bt
   - Zipline、QuantConnect、QuantLib 作为扩展阅读

这样安排有三个优点：

- 更贴近中国市场与中文教学环境
- 学生更容易完成课程作业
- 工具之间层次清楚，不会把“数据获取、作图、绩效分析、回测平台”混为一谈

### 6. 一句话总结

对于这门以中国市场为主、兼顾金融专业性与教学易用性的 Dashboard 课程，最推荐的工具组合是：

- **数据获取：Tushare、AkShare**
- **数据分析与可视化：pandas、matplotlib、seaborn、plotly**
- **金融绩效分析：QuantStats、Pyfolio、Empyrical、Alphalens**
- **回测框架：Backtrader、bt**
- **补充了解：Zipline、QuantConnect、QuantLib**
- **暂不作为重点：FinRL**

## 设计原则与最佳实践

### 1. 目标明确

一个好的 Dashboard 首先要回答“它是给谁看的、要解决什么问题”。

例如：

- 给投资经理看：重点应放在收益、回撤、风险暴露、行业配置
- 给银行业务部门看：重点应放在客户结构、地区分布、产品渗透率
- 给政府或企业管理者看：重点应放在宏观指标、行业趋势、区域比较

因此，Dashboard 设计要坚持“问题导向”，而不是“图表堆砌”。

### 2. 数据质量

金融 Dashboard 的可信度，首先来自数据质量。需要特别注意：

- 数据来源是否可靠
- 指标口径是否一致
- 时间频率是否统一
- 缺失值、异常值是否处理
- 名义值与实际值、存量与增量是否区分

在教学中，应强调“先有干净数据，再谈可视化”。

### 3. 可视化清晰

清晰比炫酷更重要。常见原则包括：

- 图表类型与变量类型匹配
- 控制颜色数量，突出重点
- 时间序列优先用折线图
- 结构比较优先用条形图或堆叠图
- 分布展示可用直方图、核密度图、箱线图
- 避免过度装饰与 3D 图形

### 4. 交互性

交互式 Dashboard 能提升使用体验，例如：

- 时间区间筛选
- 指标切换
- 行业或地区下钻
- 鼠标悬停显示明细
- 动态排序与联动更新

教学中可以用 `plotly`、`streamlit` 或 `dash` 作为简单示范工具。

### 5. 响应式设计

如果 Dashboard 需要在网页、平板、手机或会议投屏中展示，就要考虑响应式设计：

- 版面是否适配不同尺寸屏幕
- 文字和图例是否足够清晰
- 图表数量是否适中
- 关键指标是否优先展示在上方
- 是否支持打印或截图汇报

对于面向实际业务的 Dashboard，响应式与可汇报性同样重要。
