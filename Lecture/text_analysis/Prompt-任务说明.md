### Sec 2：`## 词频统计与关键词提取`

#### 2.1 `### 词频统计的局限`
- 去掉停用词后，「公司」「企业」「发展」仍然高频但缺乏区分度
- 引出问题：需要一种方法识别「在本文档中重要、但在其他文档中不常见」的词

#### 2.2 `### TF-IDF 的直觉`
- TF（词频）：这个词在本文中出现了多少次？
- IDF（逆文档频率）：这个词在所有文档中有多常见？越常见，权重越低
- TF-IDF = TF × IDF
- 金融类比：一家公司年报中反复出现「芯片」，而其他公司很少提及 → "芯片"的 TF-IDF 值很高
- 重点是直觉理解，不需要展开数学公式

::: {.callout-note}
**BM25：TF-IDF 的改进版**

如果你未来从事信息检索类工作（如构建研报搜索系统），可以了解 BM25——它在 TF-IDF 基础上加入了文档长度归一化，在实际应用中表现更稳定。Python 中可使用 `rank_bm25` 包。这里不展开，知道有这个方向即可。
:::

#### 2.3 `### Python 实现`

### Sec 3：`## 情感分析入门`

#### 3.1 `### 什么是情感分析？`
- 判断一段文本是正面、负面还是中性
- 金融场景的应用：研报看多还是看空？新闻是利好还是利空？年报语调乐观还是保守？
- 学术背景：简要提及 Loughran & McDonald (2011) 在金融文本情感分析领域的开创性工作

#### 3.2 `### 方式一：AI 工具做情感分析`
- 适用场景：几十到几百条文本，需要语境理解
- 提供完整的 AI 提示词模板（评分标准：-1 到 +1）
- 演示数据：10 条模拟券商研报摘要（从 `data_raw/analyst_reports.csv` 中读取，含明确看多、看空、中性三类）
- 展示提示词 → 展示预期返回结果格式

#### 3.3 `### 方式二：基于词典的情感分析`
- 原理：准备正面词表和负面词表，统计词数
- 情感得分 = (正面词数 - 负面词数) / 总词数
- 在代码中直接定义简单的金融情感词典（各约 20-30 个词）
- 金融情感词典简介：
  - Loughran-McDonald 词典（英文金融领域标准）
  - 中文场景：本节使用手工定义词表演示

::: {.callout-warning}
**词典法最常见的陷阱：否定词 + 正面词**

「不稳健」「无增长」「未改善」——这类表达中，正面词（「稳健」「增长」「改善」）会被词典法误判为正面情感，但实际含义恰恰相反。

简单的修正思路：在统计词频之前，用正则表达式检测「不/无/未/非/没」等否定前缀，将其后的正面词标记为负面词。示例代码：

```python
import re
# 检测并翻转否定 + 正面词的情感极性
negation_pattern = re.compile(r'(不|无|未|非|没有?)(稳健|增长|改善|向好|乐观)')
text_modified = negation_pattern.sub(lambda m: '【负面】', text)
```

这在金融文本中尤其重要，券商研报中的措辞往往十分谨慎，大量使用否定句式。
:::

- 完整代码：对 10 条研报摘要做词典法情感分析
- 输出结果表格：report_id、AI 情感得分、词典法情感得分、两种方法是否一致

::: {.callout-note}

#### 3.4 `### 两种方法的对比与讨论`
- 用表格对比 AI 方法 vs 词典方法的优劣
- 词典法局限：否定词误判、无法理解语境
- AI 方法局限：成本高、不完全可复现、透明度低
- 结论：什么场景用哪种方法更合适？
