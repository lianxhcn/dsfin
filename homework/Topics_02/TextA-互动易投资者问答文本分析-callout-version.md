## 文本分析作业（A）：互动易投资者问答的情感与主题分析

> **作业性质**：小组作业（建议 3-4 人）  
> **完成时间**：两周  
> **工具**：Python，Jupyter Notebook  
> **提交方式**：GitHub 仓库链接 + Quarto Book（GitHub Pages）

---

### 研究背景

深交所互动易平台是上市公司与投资者的官方沟通渠道。投资者提问折射出市场情绪与关切，公司回应则代表管理层的信息披露策略。当同一行业的公司面对相似的市场环境时，不同公司的投资者情绪和公司回应质量之间的差异，可能蕴含着有价值的信息信号。

本作业要求你爬取指定行业的多家公司互动易问答数据，综合运用文本清洗、情感分析、主题建模等方法，比较分析不同公司的投资者关切与信息披露特征，并初步探索文本信号与股价表现的关联。

---

### 行业选择（二选一）

**选项 A：汽车行业**

选取以下两类公司各 1-2 家进行对比（合计 3-4 家）：

- 新能源汽车代表：如比亚迪（002594）、赛力斯（601127）、理想汽车（若有A股）
- 传统汽车代表：如上汽集团（600104）、广汽集团（601238）、长城汽车（601633）

研究角度建议：新能源转型期投资者最关心什么？传统车企和新能源车企在信息披露策略上有何差异？

**选项 B：房地产行业**

选取以下两类公司各 1-2 家进行对比（合计 3-4 家）：

- 国有房企代表：如保利发展（600048）、华润置地（若有 A 股）、招商蛇口（001979）
- 民营房企代表：如万科 A（000002）、金地集团（600383）等（注意剔除已退市或停牌公司）

研究角度建议：债务危机背景下，国有房企和民营房企的投资者情绪有何差异？公司回应策略是否有系统性差异？

### 第一部分：数据爬取

**爬取范围**：每家公司爬取最近 **90 天**的全部问答记录（含未回复问题）。notes：具体能获取多少天的数据取决于该公司问答的频率和数据可得性，目标是至少 300 条记录/公司。若作业中途遇到数据获取限制，可以适当调整爬取时间范围，但需在报告中说明。

**目标字段**：问题编号、提问时间、提问内容、是否已回复、回复时间、回复内容

**数据来源**：深交所互动易（`irm.cninfo.com.cn`）

**建议使用的包**：`requests`、`pandas`、`time`、`json`

::: {.callout-tip}
### 提示词框：爬虫开发
互动易是动态渲染页面，数据通过 Ajax 接口加载，不在初始 HTML 中。给 AI 的提示词应包含以下信息：

1. 目标 URL 和你在 DevTools → Network → XHR 中找到的真实 API 接口地址
2. 该接口的请求方式（GET/POST）、请求头（User-Agent 等）、请求参数（股票代码、页码等）
3. 返回 JSON 的结构（把 `data.list[0]` 的键名复制给 AI）
4. 你需要的字段名称和翻页逻辑
5. 明确要求：加入请求间隔（`time.sleep`）、异常处理、下载日志

让 AI 先解释思路再生成代码，不要一步要求"直接给我完整爬虫"。
:::

::: {.callout-warning}
### 避坑指南：爬虫常见问题
- **翻页边界**：API 通常返回 `total` 字段表示总条数，自行计算总页数，到最后一页时停止——不要依赖"返回空列表"作为停止条件，否则可能多请求几十次空页
- **orgId 的构造**：互动易的每家公司有独立的 `orgId`，通常格式为 `gssz` + 股票代码（深交所）或 `gssh` + 股票代码（上交所），但也有例外，建议在浏览器中打开该公司页面后从 URL 中直接读取
- **请求频率**：每次请求间隔建议 2-3 秒，批量翻页时适当增加；不要用多线程并发请求
- **数据持久化**：每爬完一家公司立即保存到本地文件，不要等全部爬完再保存——爬虫中途崩溃会丢失所有数据
:::

### 第二部分：文本清洗与预处理

**建议使用的包**：`re`、`jieba`、`pandas`

清洗流程须按顺序执行，并在 Notebook 中记录每步清洗前后的文本示例对比：

1. **去除噪声**：HTML 标签、URL、表情符号、多余空白符、重复标点
2. **分词**：使用 `jieba` 进行中文分词；建议为金融领域自定义词典（如"新能源"不应被拆为"新"+"能源"）
3. **去停用词**：加载哈工大或中科院停用词表，同时自行添加金融问答场景的无效词（如"您好"、"谢谢"、"公司"、"请问"等）
4. **文本长度过滤**：分词后不足 5 个词的极短文本（通常是无实质内容的礼貌用语）可视情况剔除或单独处理

::: {.callout-tip}
### 提示词框：自定义词典
金融领域有大量专有名词会被 jieba 错误切分（如"去杠杆"被切为"去"+"杠杆"，"市盈率"被切为"市"+"盈率"）。给 AI 的提示词应包含：

- 你的行业（汽车/房地产）
- 10-20 个你预计会频繁出现的专业词汇
- 要求 AI 生成一个金融+行业领域的自定义词典文件，格式为每行一个词

然后用 `jieba.load_userdict()` 加载。
:::

::: {.callout-warning}
### 避坑指南：分词质量
- jieba 的默认词典对金融新词覆盖不足，分词质量直接影响后续所有分析结果。建议在正式分析前，随机抽取 20-30 条文本，人工检查分词是否合理
- 投资者问答中有大量数字（如"上涨 30%"、"2024 年"），在情感分析时通常是噪声，但在主题建模时可能有意义——需提前决定是否保留数字，并在报告中说明
:::

### 第三部分：描述性文本统计

在建模之前，先通过统计和可视化充分"认识"数据：

**3.1 基本统计量**

- 各公司的问答总数、已回复数、回复率、平均回复时长（小时）
- 各公司提问和回复的平均字符数分布（箱型图）

**3.2 时间分布**

- 问答数量的月度/周度时序图（各公司叠加对比）
- 是否有明显的峰值时期？结合公司重大事件（业绩公告、重大诉讼、股价异动）进行解释

**3.3 高频词与词云**

- 分公司绘制投资者提问的词云（用 `wordcloud` 包）
- 对比不同公司词云的差异：核心关键词是否不同？

::: {.callout-tip}
### 提示词框：词云设计
词云是探索性分析的直观工具，但默认词云视觉效果通常较差。提示 AI 时可以要求：生成遮罩形状（如公司 Logo 轮廓）、指定配色方案（如使用公司品牌色）、限制最大词数（50-80 个词比 200 个词更清晰）。
:::

### 第四部分：情感分析

**建议使用的包**：`snownlp`、`transformers`（选做）、`pandas`、`matplotlib`/`plotly`

**方法一（必做）：词典法**

使用 SnowNLP 对每条文本打分（0-1，越高越正面），或使用金融领域情感词典（NTUSD、Loughran-McDonald 中文版）计算正负词比例。

::: {.callout-tip}
### 提示词框：情感词典法
给 AI 的提示词应说明：
- 你使用的词典名称和格式（正面词列表 + 负面词列表）
- 你希望的打分公式（如：正面词数 / 总词数 − 负面词数 / 总词数）
- 是否考虑否定词的处理（如"不满意"应算负面，但简单词典法会漏掉）
- 要求 AI 解释词典法的局限性，以及什么情况下结果不可信
:::

**方法二（选做）：预训练模型**

使用 FinBERT 或 ERNIE-Finance 等金融领域预训练模型，对文本做三分类（正面/中性/负面）。

::: {.callout-warning}
### 避坑指南：情感分析结果验证
- SnowNLP 基于电商评论训练，对金融问答的适用性有限。建议人工抽样验证：随机选取 30 条问题文本，人工标注情感，与模型结果对比，计算准确率
- 投资者问答中存在大量"中性"陈述（如"请问公司的资产负债率是多少"），这类文本用任何方法打分都意义不大——考虑只对包含明显情感词的文本做情感分析
- 词典法和模型法的结果相关性通常不高，这不一定意味着其中一个错了，两者衡量的维度本来就有差异
:::

**分析输出**：

- 各公司投资者提问的情感得分分布（箱型图）
- 情感得分的月度时序趋势（各公司对比折线图）
- 投资者情绪是否与股价同期走势有关联？（散点图 + 相关系数，此处不要求因果推断）

### 第五部分：主题建模（LDA）

**建议使用的包**：`gensim`、`pyLDAvis`、`matplotlib`

**5.1 建模流程**

对投资者提问文本（建议分公司分别建模，再做对比）进行 LDA 主题建模，确定最优主题数。

::: {.callout-tip}
### 提示词框：LDA 主题建模
给 AI 的提示词应包含：
- 你已完成的预处理步骤（分词、去停用词后的词袋格式）
- 你希望尝试的主题数范围（如 3 到 10）
- 要求 AI 解释如何使用 Coherence Score（相干性得分）选择最优主题数，并绘制相干性得分随主题数变化的折线图
- 要求使用 `pyLDAvis` 生成交互式主题可视化，并嵌入 Quarto 报告
- 明确要求 AI 解释每个主题的含义（输出每个主题的 Top 10 关键词后，让 AI 帮你解读经济含义）
:::

**5.2 主题对比分析**

- 不同公司的主题分布是否有差异？（各公司文档的主题权重均值对比）
- 同一主题在不同公司的代表性文本是否相似？
- 主题比例是否随时间变化？（如房地产行业"债务偿还"主题在 2022-2023 年是否上升）

::: {.callout-warning}
### 避坑指南：LDA 常见问题
- LDA 对预处理质量极敏感。如果停用词清理不彻底（如"公司"、"请问"未去除），这些词会主导每个主题，导致主题毫无区分度
- Coherence Score 选出的最优主题数不一定是最有解释意义的。建议在统计最优附近的 2-3 个方案都试一试，选择主题含义最清晰、最容易解释的那个
- 样本量太少（如每家公司只有 200 条问题）时 LDA 效果很差；90 天数据若不足 300 条，建议换成 180 天
:::

### 第六部分：公司回应质量分析

这部分分析的是**公司回复文本**（而非投资者提问），从信息披露质量角度切入：

**建议分析维度**：

- **回复率与回复速度**：不同公司的回复率和平均回复时长是否有系统性差异？
- **回复长度**：更长的回复是否意味着更高质量？还是更多套话？（需人工抽样验证）
- **回复情感**：公司回复的语气（积极/消极/中性）是否因公司、时期不同而有差异？
- **信息含量评估**：对回复文本进行 LDA 建模，是否能识别出"实质性回复"和"模板化回复"两类主题？

::: {.callout-tip}
### 提示词框：信息含量评估
识别"套话式回复"与"实质性回复"是一个有趣的文本分类问题。可以给 AI 以下提示：
- 描述两类回复的典型特征（套话：感谢关注、以公告为准；实质性：提供具体数字、说明具体时间节点）
- 要求 AI 设计一个简单的规则分类器（如基于特定关键词）来区分两类回复，并统计各公司的实质性回复比例
- 这不需要训练机器学习模型，一个精心设计的关键词规则通常就足够了
:::

### 第七部分：文本信号与股价的初步关联（选做）

::: {.callout-note}
本部分为选做内容，不计入基础分，完成可获额外加分。不要求因果推断，只需呈现相关性的描述性证据。
:::

**思路框架**：

1. 将情感得分聚合为周度或月度均值（`sentiment_weekly`）
2. 使用 akshare 获取对应公司同期的股价数据，计算周度或月度收益率
3. 绘制情感指数与股价收益率的时序对比图（双轴图）
4. 计算两者的 Pearson 相关系数（同期及滞后 1-4 周）

::: {.callout-warning}
### 避坑指南：相关性分析的陷阱
- 情感得分与股价收益率通常都存在时序自相关，直接计算 Pearson 相关系数可能产生伪相关。建议先对两个序列分别做一阶差分，再计算相关系数
- 样本量（通常只有 12-16 个月度观测）太少，任何相关性结论都不稳健，务必在报告中说明这一局限性
- 相关性不等于因果性——如果发现显著相关，讨论可能的解释时不要用"情感影响股价"这样的因果语言，而应说"两者共同受到某些市场事件的驱动"
:::
