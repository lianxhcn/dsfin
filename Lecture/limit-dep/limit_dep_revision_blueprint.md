# Limit-dep 模块修订蓝图

> 本文档用于统一 `04_tobit_model`、`05_truncated_twopart` 和 `06_heckman` 三章的案例、术语、模型边界和代码风格。当前版本暂不处理 slides，待讲义定稿后再统一生成。

## 1. 核心写作原则

本模块不把模型选择简化为“因变量有很多 0 就用某个模型”。更稳妥的顺序是：

1. **先判断样本是否进入数据**：如果不满足条件的样本根本没有进入数据，则是截断问题。
2. **再判断因变量是否完整可观测**：如果样本在数据中，但边界外取值被统一记录为边界值，则是归并问题。
3. **再判断 0 的经济含义**：如果 0 是真实经济选择，则 Tobit 的共同潜在机制可能太强。
4. **最后判断结果变量是否 missing**：如果结果只在被选择样本中可见，则进入 Heckman selection。

后续三章都围绕企业信贷背景展开，但每章从不同的数据生成机制切入。这样可以帮助学生理解：模型选择不是换一个故事，而是对同一个经济现象提出不同的观测机制解释。

## 2. 术语规范

| 英文术语 | 主要中文译法 | 说明 |
|---|---|---|
| `censoring` | 归并 | 样本仍在数据中，但边界外的潜在取值被统一记录为边界值。中文文献中也常见“删失”“审查”“截尾”等译法，但本讲义尽量不用“截尾”指代 `censoring`。 |
| `truncation` | 截断 | 不满足条件的样本没有进入数据。 |
| `corner solution` | 角点解 | 0 是经济主体真实选择的结果。 |
| `sample selection` | 选择性观测 / 样本选择 | 结果变量只在通过某个选择机制后才可观测。 |
| `structural zero` | 结构性零值 | 个体在机制上不可能产生正值。 |
| `incidental zero` | 随机零值 | 个体可能产生正值，但当期实现为 0。 |

## 3. 统一案例：企业信贷数据

三章共享同一个大背景：企业与银行信贷关系。但每章研究的问题不同。

| 章节 | 研究问题 | 被解释变量 | 关键观测机制 |
|---|---|---|---|
| 04 Tobit | 企业银行贷款金额如何决定 | `loan_amt` | 潜在净借款需求 $B_i^*$ 被归并到 0 |
| 05 Two-part 等 | 企业是否贷款以及贷多少 | `loan_amt` | 是否获得贷款与获贷后金额是两个机制 |
| 06 Heckman | 企业贷款利率如何决定 | `loan_rate` | 没有贷款的企业没有贷款利率，结果变量不可观测 |

扩展模型也尽量围绕信贷背景：

| 模型 | 信贷场景中的例子 |
|---|---|
| Hurdle model | 某些项目贷款或债券发行存在最低经济规模，企业先决定是否跨过门槛 |
| Zero-inflated model | 贷款笔数为 0 可能来自结构性无信贷需求，也可能来自普通计数过程中的随机 0 |
| Fractional response model | 银行贷款占总负债比例、绿色贷款占总贷款比例 |

## 4. 三章边界

### 4.1 04 Tobit

04 章的主线是：当企业存在一个潜在净借款需求 $B_i^*$，而实际银行贷款金额不能为负时，$B_i^*\le 0$ 的企业都被记录为 $B_i=0$。Tobit 的核心不是“处理很多 0”，而是把 0 和正值金额看作同一个潜在连续机制的不同表现。

主设定为：

$$
B_i^*
=
\alpha
+\beta_1 opportunity_i
+\beta_2 collateral_i
-\beta_3 cash_i
+u_i
$$

$$
B_i=\max(0,B_i^*)
$$

这里 $B_i^*<0$ 不表示“负贷款”，而表示企业没有使用银行贷款的净动机。

### 4.2 05 Two-part、Hurdle、Zero-inflated 与 FRM

05 章的主线是：如果 0 是真实经济选择，或者结果变量有自然边界，Tobit 的共同潜在机制就可能过强。

Two-part 主设定为：

$$
D_i=1(B_i>0)
$$

$$
P(D_i=1\mid z_i)
=
F(\gamma_0+\gamma_1 collateral_i+\gamma_2 bank\_access_i)
$$

$$
E(B_i\mid B_i>0,x_i)
=
\exp(\beta_0+\beta_1 collateral_i+\beta_2 opportunity_i)
$$

其中 `collateral` 是公共变量，`bank_access` 主要影响能否获得贷款，`opportunity` 主要影响获贷后贷多少。

### 4.3 06 Heckman

06 章的主线是：没有贷款的企业，其贷款利率不是 0，而是不可观测。因此，研究贷款利率时不应使用 Two-part 的 0 结果逻辑，而应考虑选择性观测。

简化设定为：

$$
D_i^*
=
\gamma_0
+\gamma_1 collateral_i
+\gamma_2 bank\_access_i
-\gamma_3 risk_i
+v_i
$$

$$
D_i=1(D_i^*>0)
$$

$$
rate_i
=
\beta_0
-\beta_1 collateral_i
+\beta_2 risk_i
+u_i,
\quad D_i=1
$$

其中 `bank_access` 是选择方程中的排他性变量候选。它是否合理，取决于经济机制与制度背景，而不是统计显著性本身。

## 5. 独立性要求

虽然 04、05、06 彼此相连，但每章都应能被单独讲授。因此：

- 每章开头都重新交代本章最小背景；
- 正文不假设学生已经完整读过前一章；
- 跨章比较尽量放在 callout 或结尾小表中；
- 每章都明确说明本章的 $y$、潜变量、观测规则和核心 $x$ 变量。

## 6. 代码与图形规范

三章代码统一使用以下基础变量：

| 变量 | 含义 |
|---|---|
| `opportunity` | 投资机会或资金需求强度 |
| `collateral` | 抵押能力或可抵押资产比例 |
| `bank_access` | 银行服务可得性 |
| `cash` | 内部资金充裕程度 |
| `risk` | 企业风险 |
| `loan_amt` | 银行贷款金额 |
| `loan_rate` | 银行贷款利率 |
